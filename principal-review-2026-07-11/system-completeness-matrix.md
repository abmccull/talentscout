# TalentScout System-Completeness Matrix

This matrix traces each major player-facing system to the store, engine and durable state. A screen or type is not treated as implementation unless its outputs reach persistent state and affect later processing.

## Legend

**Completeness score**

- **5 — Complete and meaningful:** authoritative, persistent, consequential and well tested.
- **4 — Complete and meaningful with contained gaps:** core loop is authoritative; secondary depth or feedback is missing.
- **3 — Functional but shallow:** state changes and persists, but decisions/consequences are thin.
- **2 — Partially connected:** important outputs are missing, discarded or bypassable.
- **1 — UI ahead of engine / engine ahead of UI / placeholder or misleading.**
- **0 — Broken:** cannot be trusted for its stated purpose.

**Test evidence**

- **U✓** — included in the 20 currently passing Vitest tests.
- **E✓** — specifically exercised in a Playwright run that completed during this review.
- **E?** — an E2E spec exists, but the full suite did not complete and no passing result is claimed for that case.
- **—** — no focused automated test found.

Final typecheck and production build passed. The full E2E run nevertheless encountered two reproducible failing cases and stopped before completing the 235-test inventory, so an E2E file's existence is not evidence that it currently passes.

## Cross-system conclusions

### Authoritative state paths

The strongest connected path is:

`Calendar activity → manual week/day resolution → completed observation → scouting knowledge/journal → player dossier`.

Even this path is compromised by count inflation and transient session state, but its player-facing cause and effect is recognizable.

The weakest paths are:

- `Quick advance → world history` — not equivalent to manual time.
- `Rival bid → transfer` — message without transaction.
- `Standings → promotion/relegation → next season` — display without membership change.
- `Hypothesis → new evidence → revised belief` — durable text without engine resolution.
- `Report → stakeholder need → club decision → fit outcome` — report exists, but no durable case/brief.
- `Steam save → later load/conflict/delete` — write without provider lifecycle.

### Completeness distribution

No reviewed system currently earns an unqualified 5. The four-score systems—calendar scheduling, profiles, free agents, youth discovery, equipment and NPC delegation—still depend on broken world, evidence or persistence foundations. Eleven systems are rated 0 or 1 and should be considered non-authoritative until fixed or hidden.
